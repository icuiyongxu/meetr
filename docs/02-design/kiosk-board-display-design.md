# 大屏/看板模式（Kiosk / Board Display）设计说明

## 1. 背景

MRBS 的 `kiosk.php` 提供一种**密码锁定模式**，让用户在公共电视/大屏上只能看日历，不能操作。

对 Meetr 而言，更适合做一个**纯展示型看板页面**：
- 无需登录，任何人都能看
- 大字高对比度，适合大厅/会议室电视
- 显示当前会议室当天所有预约
- 当前会议 + 即将开始的会议高亮
- 自动刷新，无需人工操作

---

## 2. 目标

1. 独立路由 `/kiosk`，无需登录
2. 大屏友好：大字、高对比度、宽可视角度
3. 支持全楼栋所有会议室轮播展示
4. 支持指定单会议室展示
5. 当前正在进行的会议高亮
6. 自动刷新（默认 30 秒）

---

## 3. 页面结构

### 3.1 全局看板（`/kiosk`）
- 顶部：大标题 "会议室看板"
- 楼栋选择标签
- 下方：选中楼栋的所有会议室卡片，横向排列
- 每张卡片：会议室名 + 当天预约时间轴

### 3.2 单会议室看板（`/kiosk/:roomId`）
- 全屏单会议室模式
- 显示该会议室当天所有预约
- 大字展示：当前会议 / 下一场会议
- 无导航，适合单独投屏

### 4.2 颜色规则
- 当前会议（进行中）：绿底白字，高亮
- 即将开始（15分钟内）：黄底
- 已结束：灰色
- 未开始：白底蓝字

### 4.3 大屏友好样式
- 最小字号：16px（内容）~ 48px（标题/时间）
- 字体加粗
- 高度对比
- 无需鼠标 hover，所有内容一览无余

## 5. 后端设计

### 5.1 新增接口

#### 获取看板数据
`GET /api/kiosk/rooms/:roomId/days/:dateMs/bookings`

返回：
- 会议室基本信息（名称、楼栋）
- 当天所有预约（排序）
- 当前会议（如果有时）

#### 批量获取多会议室（全局看板用）
`GET /api/kiosk/buildings/:buildingId/days/:dateMs`

返回：
- 楼栋下所有会议室 + 每个会议室的当天预约

### 5.2 接口说明
- **无需登录**，所有人都能访问
- 所有参数：roomId / buildingId / dateMs（epoch ms）
- 预约按开始时间升序

### 5.3 返回字段
```json
{
  "room": { "id": 1, "name": "301会议室", "buildingName": "3号楼" },
  "date": "2026-03-28",
  "bookings": [
    {
      "id": 1,
      "subject": "产品评审会",
      "bookerName": "张三",
      "startTimeMs": 1741574400000,
      "endTimeMs": 1741578000000,
      "status": "IN_PROGRESS",  // IN_PROGRESS | UPCOMING | ENDED
      "isCurrent": true
    }
  ]
}
```

## 6. 前端设计

### 6.1 路由
- `/kiosk` — 全局看板（楼栋选择 + 多会议室卡片）
- `/kiosk/room/:roomId` — 单会议室看板

### 6.2 全局看板 `/kiosk`
- 顶部：楼栋选择标签（横向滚动）
- 下方：会议室卡片网格（响应式，TV 上 3-4 列）
- 每张卡片显示：会议室名 + 当天预约时间轴（精简）
- 点击卡片 → 跳转 `/kiosk/room/:roomId`

### 6.3 单会议室看板 `/kiosk/room/:roomId`
- 全屏布局，无导航栏
- 顶部：会议室名 + 日期 + 时钟
- 主区域：当前/下一场会议大字展示
- 下方列表：当天剩余预约（时间 + 主题）
- 底部：自动刷新倒计时

### 6.4 自动刷新
- 默认 30 秒轮询
- 显示倒计时，进度条
- 可手动点击刷新

- 后端接口

---

## 8. 技术约定

- 时间格式：统一用 `datetime.ts` 工具函数
- 时间字段：epoch ms
- 后端无需权限校验
- 响应式优先考虑 TV（1920x1080）+ 移动端

---

## 9. 验收标准

1. 访问 `/kiosk/room/1` 无需登录，显示会议室 1 当天预约
2. 当前会议（时间吻合）高亮绿色
3. 页面大字清晰，TV 3米外可读
4. 30 秒自动刷新，倒计时可见
5. 刷新页面数据最新
