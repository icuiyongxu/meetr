# 重复预约（系列预约）管理补全测试方案

> 对应功能：系列详情、系列修改范围、本次/后续/全部取消、冲突校验、系列日志记录

---

## 1. 测试前置条件

1. 数据库已初始化
2. 至少存在一个会议室
3. 已开启审批规则（可选）
4. 已有普通用户账号 + 管理员账号

---

## 2. 功能用例

### 用例 A：创建系列预约

**步骤**
1. 普通用户进入会议室日历
2. 新建预约
3. 选择重复规则（每周 / 每天 / 工作日 / 每月）
4. 设置结束日期
5. 提交预约

**预期**
- 创建成功
- 系列内生成多条 booking
- “我的预约”列表中显示系列标识

---

### 用例 B：查看系列详情

**步骤**
1. 在“我的预约”中点击某条系列预约
2. 点击“系列”按钮

**预期**
- 弹出系列详情
- 展示主预约 + 所有子预约
- 每条显示时间、状态、审批状态

---

### 用例 C：修改本次

**步骤**
1. 选择一条系列预约
2. 修改时间或主题
3. 选择“仅修改本次”

**预期**
- 只有当前实例被修改
- 其他实例保持不变
- 系列日志新增 UPDATE_ONCE

---

### 用例 D：修改本次及后续

**步骤**
1. 选择一条系列预约
2. 修改时间或主题
3. 选择“修改本次及后续”

**预期**
- 当前实例及之后实例被修改
- 之前实例保持不变
- 系列日志新增 UPDATE_FUTURE

---

### 用例 E：修改整个系列

**步骤**
1. 选择一条系列预约
2. 修改时间或主题
3. 选择“修改整个系列”

**预期**
- 系列所有实例被修改
- 系列日志新增 UPDATE_ALL

---

### 用例 F：取消本次

**步骤**
1. 选择一条系列预约
2. 点击取消
3. 选择“仅取消本次”

**预期**
- 只有当前实例状态变为 CANCELED
- 其他实例保持不变
- 系列日志新增 CANCEL_ONCE

---

### 用例 G：取消本次及后续

**步骤**
1. 选择一条系列预约
2. 点击取消
3. 选择“取消本次及后续”

**预期**
- 当前实例及后续实例状态变为 CANCELED
- 之前实例保持不变
- 系列日志新增 CANCEL_FUTURE

---

### 用例 H：取消整个系列

**步骤**
1. 选择一条系列预约
2. 点击取消
3. 选择“取消整个系列”

**预期**
- 系列全部实例状态变为 CANCELED
- 系列日志新增 CANCEL_ALL

---

### 用例 I：冲突校验

**步骤**
1. 人为制造冲突时间段
2. 修改本次/后续/全部

**预期**
- 修改失败
- 返回冲突列表
- 前端提示冲突预约

---

## 3. 接口测试点（可选）

### 系列详情
```
GET /api/bookings/{id}/series
```

### 系列修改
```
PUT /api/bookings/{id}/series
```

### 系列取消
```
POST /api/bookings/{id}/series-cancel
```

---

## 4. 日志验证

检查 booking_operation_log：
- 是否记录 scope
- 是否记录操作类型
- 是否记录操作人
- 是否记录时间
