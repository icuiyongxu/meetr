# Meetr 产品需求草案

## 1. 项目背景
现有 MRBS 已验证会议室预定场景的核心业务模型，但技术栈老旧，不适合继续作为现代企业内部系统的长期底座。Meetr 项目目标是在保留成熟业务规则的前提下，使用 Spring Boot + Vue3 + Element Plus 重写一版新的会议室预定系统。

## 2. 产品目标
- 提供清晰易用的会议室预约体验
- 支持管理端维护会议室、规则和状态
- 保证预约冲突校验准确可靠
- 支持后续扩展审批、通知、签到、统计

## 3. 用户角色
### 3.1 普通用户
- 查询空闲会议室
- 创建/修改/取消自己的预约
- 查看我的预约和今日会议

### 3.2 管理员
- 维护园区/楼栋/会议室
- 配置预约规则
- 处理预约异常和强制操作

### 3.3 系统管理员
- 管理全局参数
- 配置审批/通知扩展能力

## 4. 一期功能范围
### 4.1 主数据管理
- 园区/地点
- 楼栋/楼层
- 会议室
- 容量、设备、状态、描述

### 4.2 预约功能
- 空闲会议室查询
- 创建预约
- 修改预约
- 取消预约
- 查看预约详情
- 时间冲突校验

### 4.3 我的工作台
- 我的预约
- 今日会议
- 即将开始会议

### 4.4 管理能力
- 会议室管理
- 预约规则配置
- 房间状态启停

## 5. 核心业务规则
1. 同一会议室同一时间段不得重叠预约
2. 开始时间必须早于结束时间
3. 不能预约过去时间
4. 停用会议室不可创建新预约
5. 取消预约后立即释放时间占用
6. 一期默认审批中预约也占用时间段

## 6. 暂不纳入一期
- 周期性预约
- 小程序/App
- 门禁/硬件联动
- 高级统计报表
- 多租户复杂能力

## 7. 成功标准
- 普通用户可在 1 分钟内完成一次预约
- 管理员可独立维护会议室数据
- 不出现双重预约
- 支持后续平滑扩展审批与通知能力
